# 内网穿透(校园网)

## 利用tailscale实现内网隔离下的ssh连接

### tailscale原理(Gemini生成)

> 我的理解是，在校园网内另外搭建了一个私有局域网，并打破了正常校园网内的内网隔离(直接ssh/ping均不成功，但tailscale可以)。同一tailscale账户下的设备可以互相访问，而校园网内其他设备无法访问。

Tailscale 的核心目标是**轻松地在任何地方的设备之间创建安全、私密的网络连接**，就像它们在同一个局域网中一样，而无需手动配置防火墙或复杂的 VPN 设置。它主要基于以下原理：

1.  **WireGuard® 基础:** Tailscale 底层使用现代、高性能的 VPN 协议 **WireGuard** 来建立设备之间的加密隧道。WireGuard 以其速度快、配置简单和安全性高而闻名。

2.  **协调服务器 (Coordination Server):** 这是 Tailscale 的“大脑”，但**通常不处理您的网络流量**。它的主要作用是：
    *   **身份验证:** 通过您选择的身份提供商（如 Google, Microsoft, GitHub, Okta 等）验证用户和设备。
    *   **公钥分发:** 安全地交换连接所需设备之间的 WireGuard 公钥。
    *   **IP 地址管理:** 为每个加入网络的设备分配一个唯一的、稳定的私有 IP 地址（通常在 `100.x.y.z` 网段）。
    *   **NAT 穿透协调:** 帮助设备找到彼此并尝试建立直接连接。

3.  **NAT 穿透 (NAT Traversal):** 这是 Tailscale 的关键技术之一。大多数设备都位于 NAT（网络地址转换）设备或防火墙之后，没有公共 IP 地址，直接连接很困难。Tailscale 使用 **STUN** 和 **ICE** 等技术来帮助设备“打洞”(UDP hole punching)，发现彼此的公共 IP 地址和端口，并尝试建立**点对点 (Peer-to-Peer, P2P)** 的直接 WireGuard 连接。

4.  **点对点连接 (P2P):** 如果 NAT 穿透成功，设备之间会建立直接的、加密的 WireGuard 隧道。这是**最理想、速度最快**的连接方式，因为数据直接在设备间传输，不经过任何中间服务器。

5.  **中继服务器 (DERP):** 如果由于严格的防火墙或对称 NAT 等原因，NAT 穿透失败，无法建立直接 P2P 连接，Tailscale 会使用其全球分布的 **DERP (Designated Encrypted Relay for Packets)** 服务器作为**后备中继**。数据流量会通过加密的方式经由最近的 DERP 服务器转发。这确保了连接总是可用，但速度会比 P2P 连接慢，因为增加了一个中间环节。

**总结流程：**

1.  设备安装 Tailscale 客户端并登录（通过身份提供商）。
2.  客户端连接到 Tailscale 协调服务器，进行身份验证，获取私有 IP 和其他在线设备的公钥及连接信息。
3.  当一个设备想要连接另一个设备时，它们在协调服务器的帮助下，尝试使用 NAT 穿透技术建立直接的 P2P WireGuard 隧道。
4.  如果 P2P 成功，数据直接加密传输。
5.  如果 P2P 失败，数据通过加密的方式经由 Tailscale 的 DERP 中继服务器转发。

通过这种方式，Tailscale 创建了一个**覆盖网络 (Overlay Network)**，将分散在不同物理网络的设备安全地连接在一起，形成一个扁平、易于管理的虚拟私有网络。

### SSH连接及SFTP协议

**SSH(Secure Shell)**是一种网络协议，用于在不安全的网络上安全地访问计算机。它提供了强大的身份验证和加密功能，确保数据在传输过程中不被窃听或篡改。SSH 通常用于远程登录到服务器、执行命令和管理系统。

**SFTP**(SSH File Transfer Protocol，安全文件传输协议)，运行在 SSH 协议之上，通常也使用SSH TCP端口22。与不加密的**FTP(File Transfer Protocol)**、加密但需要额外端口(如990)的**FTPS(FTP over SSL/TLS)**相比，SFTP提供了更高的安全性和灵活性。

不过，SFTP 客户端提供的是一个受限的文件传输环境，而不是一个完整的远程 Shell。它只支持 SFTP 协议定义的命令（如 `ls`, `cd`, `get`, `put`, `mkdir`, `rm` 等），用于文件操作，不能执行服务器上的任意 Shell 命令（如 sudo, apt 等）。
